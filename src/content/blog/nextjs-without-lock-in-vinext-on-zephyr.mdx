## vinext: Next.js API Surface, Vite Engine

Cloudflare recently open-sourced `vinext`, a Vite-powered reimplementation of the Next.js API surface.

Built on Vite. Designed for edge runtimes. Deployable in a single command.

What this means:

- Familiar Next-style ergonomics
- Dramatically faster local builds via Vite
- Compatibility with edge-native runtimes
- Reduced dependency on vertically integrated hosting platforms

`vinext` decouples framework ergonomics from hosting assumptions.

That is the structural shift.

## Why Framework-Hosting Coupling Became a Problem

For years, deploying Next.js in production typically meant deploying to Vercel.

That pairing delivered:

- Tight integration
- Fast deployment
- Simplified developer experience

But it also introduced coupling:

- Framework changes tied to platform rollout cycles
- Runtime features dependent on specific infrastructure
- Architectural decisions constrained by vendor direction
- Cost scaling tied to a single provider

Modern engineering teams increasingly want:

- Infrastructure optionality
- Predictable upgrade paths
- Federation compatibility
- Long-term architectural control

`vinext` makes decoupling possible.

Zephyr Cloud makes it production-ready.

## vinext + Zephyr Cloud: Instant Dev to Global Edge

We paired `vinext` with Zephyr's Vite plugin and deployed a live demo.

Workflow:

- Install `vite-plugin-vinext-zephyr`
- Add `withZephyr()` to `vite.config.js`
- Build and deploy

That is it.

No CI/CD configuration. No custom edge worker scaffolding. No environment-specific rewrites.

Local build to global edge deployment in under one second.

Live demo: [https://lnkd.in/gp38k57p](https://lnkd.in/gp38k57p)

This is not theoretical performance. It is measurable deployment velocity.
