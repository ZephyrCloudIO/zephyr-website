## Summary

This release introduces new audit logging capabilities, a simpler way to trigger redeployments. It also includes several bug fixes that improve reliability and consistency across the platform.

## Audit Logs (Beta Feature)

We've introduced audit logging to provide better visibility into activity across your projects and applications.

- **Application Audit Logs** allow you to track who performed actions within an application and when those actions occurred.
- **Project Audit Logs** provide the same level of visibility for project-level activity, helping teams better understand changes across projects.

## Redeployments Flow Became Easier

A new Redeployment page makes it easier to redeploy without pushing a new version, which can be useful for retrying deployments or refreshing environments.

## Tags System Improvements

Tags have been reworked into a dedicated packages structure. This improves reliability and prepares the system for more advanced tag management workflows.

## Application UI Improvements

Application pages have been reorganized, making it easier to move between application features.

## Bug Fixes and Stability Improvements

- Environment logs filtering fixes, making it easier to find relevant entries.
- Fixed theme color inconsistencies on the subscriptions page.
- Fixed an issue where nullable version tags could not be updated.
- Fixed a bug where open membership settings were not saved correctly.
- Fixed an issue where tag values were not set correctly when creating a new tag.

## What This Means for Teams

- Better visibility into project and application activity with new audit logs.
- Easier redeployments.
- Cleaner navigation across application pages.
- Improved reliability and consistency across several areas of the platform.
