## Summary

This release focuses on getting new users to their first deployment faster with a redesigned onboarding experience, alongside performance improvements to application list loading for larger organizations.

## Redesigned Onboarding

The onboarding flow has been redesigned to reduce the steps between signing up and deploying your first application.

- The flow is shorter and more focused, guiding new users directly to the actions that matter most.
- Each step is clearer about what Zephyr needs and what happens next.
- Teams and organizations are set up with less back-and-forth.

## Faster Application List Loading

Application list queries have been optimized to run significantly faster, especially for organizations with many applications and versions. The improvement is most noticeable on the dashboard's main application and project views.

## Reliability Fixes

- Users rejoining an organization through an open membership domain are now correctly auto-promoted rather than left in a pending state.
- Several visual inconsistencies in the UI have been resolved.

## What This Means for Teams

- New users reach their first deployment faster with less friction in onboarding.
- Dashboard pages load more quickly for larger organizations.
- Open membership domain rejoins are handled correctly without manual intervention.
